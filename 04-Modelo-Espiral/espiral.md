# Modelo Espiral
Modelo de desenvolvimento de software que combina:
- Interação(como no modelo incremental)
- **Gerenciamento de riscos**

## Objetivo:
* **Reduzir riscos** no desenvolvimento
* Permitir ajustes contínuos
* Melhorar a **qualidade do produto** final

## Quando usar?
* Projetos grandes
* Alta incerteza
* Sistemas críticos
* Necessidade de validação

---

## Estrutura
Ocorre em **ciclos(espirais)**
Cada ciclo possui fases bem definidas
Foco em **análise de risco** a cada ciclo

### Fases

* Planejamento
* Análise de riscos
* Engenharia(desenvolvimento)
* Avaliação(feedback do cliente)

---

## Vantagens
* **Excelente para projetos complexos**
* Identifica riscos cedo
* Flexível a mudanças
* Envolvimento do cliente

## Desvantagens
* **Alto custo**
* Complexidade de gestão
* Não recomendado para projetos pequenos
