# Modelo Cascata

* Sequencial
* Linear
* Uma fase por vez
* Cada etapa termina antes da próxima começar

---

## Etapas
* Requisitos
* Analise
* Projeto
* Implementação
* Testes
* Manutenção

### Vantagens
* Simples de entender
* Fácil de gerenciar
* Boa documentação
* Bom para requisitos bem definidos

### Desvantagens
* Mudanças são dificeis
* Cliente vê o sistema apenas no final
* Alto risco de retrabalho
* No modelo clássico não há retorno entre fases
